Opdracht Regular Expressions: deel 1

  • De bedoeling van deze opdracht is om je eigen regular expression tester te maken. Dit ziet er ongeveer als volgt uit:

    RegEx tester

  • Je kan een regular expression en een string ingeven. Deze waarden submit je via PHP.
  • Het php-script controleert of de regular expression een match vindt.
    • Gebruik hiervoor preg_replace() en denk aan de conventie die je moet gebruiken om regular expressions in deze functie werkende te krijgen.
    • Wanneer er een match gevonden is, moet het PHP script de match(en) vervangen door een #-symbool. Dit #-symbool kan via css gestyled worden.
  • De ingevulde waarden moeten bewaard blijven.
  • Het resultaat wordt onderaan het formulier getoond. Wanneer er geen match werd gevonden, moet er een melding getoond worden.
  • Het resultaat ziet er als volgt uit:

    RegEx tester

    Resultaat: Dit is ##n t#st

Opdracht Regular Expressions: deel 2

  • Probeer je script en je regular expression kennis uit door deze voorbeeldstrings te matchen met de opdracht. Lijst je resultaten op in de formulier-pagina.
  • Match alle letters tussen a en d, en u en z (hoofdletters inclusief)
    • String: Memory can change the shape of a room; it can change the color of a car. And memories can be distorted. They're just an interpretation, they're not a record, and they're irrelevant if you have the facts.
  • Match zowel colour als color
    • String: Zowel colour als color zijn correct Engels.
  • Match enkel de getallen die een 1 als duizendtal hebben.
    • String: 1020 1050 9784 1560 0231 1546 8745
  • Match alle data zodat er enkel een reeks "en" overblijft.
    • String: 24/07/1978 en 24-07-1978 en 24.07.1978
