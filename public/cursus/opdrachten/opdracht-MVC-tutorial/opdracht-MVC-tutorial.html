  • Lees wat er over elk bestand wordt gezegd en probeer te begrijpen wat de functie is van de bestanden die je net hebt aangemaakt.

  • Extra aandacht bij het aanmaken van de database en het config.php bestand
    • In de tutorial staat dat je een database moet aanmaken met de naam 'todo'. Verander de databasenaam in opdracht_MVC_tutorial

    • Voeg aan dit config.php bestand een nieuwe constante toe: BASEPATH_HARDCODED
    • Deze constante heeft als waarde de root url naar de folder waar jouw applicatie toegankelijk is (bv. http://www.app.local/opdracht_MVC_tutorial/ )

  • Als alles goed is verlopen, kan je surfen naar http://www.app.local/opdracht_MVC_tutorial/items/viewall/
  • Test alle functies (de toevoegfunctie, het bekijken van een individuele entry en het deleten van een entry)

Opdracht MVC Anant Garg: deel 2 - breid het framework uit

  • Deze uitbreiding is een uitbreiding van de applicatie gebaseerd op de Items applicatie.
  • Wanneer je surft naar http://www.app.local/opdracht_MVC_tutorial/users/viewall/ krijg je een overzicht van alle users in de database opdracht_MVC_tutorial
  • Normaalgezien zou je na het doorlopen van de tutorial moeten weten dat je dus een tabel users zal moeten aanmaken.
  • In deze tabel zitten de volgende velden:
    • id: (primary key, unsigned, auto_increment)
    • email (varchar 255 karakters)
    • password (varchar 255 karakters)
    • last_login (datetime)
  • Baseer je op de bestaande controller, model en views uit de application folder van het Items onderdeel uit de tutorial.
  • Voor de controller van de Users application zijn vier mogelijk actions:
    • viewall(): Geeft een overzicht van alle gebruikers in de tabel users
    • view(): Toont alle data van een gebruiker (email, password en last_login) op basis van een ID
    • add(): Voegt een gebruiker toe aan de tabel users. Het paswoord moet MD5 hashed zijn
    • delete(): Verwijdert een gebruiker uit de tabel users op basis van het ID
  • Hoe deze actions worden uitgevoerd mag je zelf beslissen, maar uiteraard is het best om dezelfde analogie te volgen als de Items
  • Het model heeft dezelfde functies als het model van het Items model
  • Volgende views zijn aanwezig (zet deze in de daarvoor bestemde map -> zie tutorial)
    • header.php (naar analogie met de header van de Items).
      • Dit echoot de titel die je in de users-controller hebt geset in het titel-attribuut en in <h1> waarin de titel nogmaals voorkomt
    • footer.php
      • Dit echoot een paragraaf met je naam
    • viewall.php
      • Dit echoot een formulier dat verwijst naar de add action
      • Het gaat hier om een post, omdat er data in de database wordt ingevoerd
      • Dit echoot eveneens een overzicht van alle e-mailadressen van de gebruikers in de tabel users.
      • Elke e-mailadres is een link naar de detailpagina van de gebruiker, de view-action gevolgd door het ID van de gebruiker, bv: http://www.app.local/opdracht_MVC_tutorial/users/view/1/ )
      • Elk e-mailadres wordt gevolgd door een 'Verwijder gebruiker' link. Deze link verwijst naar de delete action, bv. http://www.app.local/opdracht_MVC_tutorial/users/delete/1/
    • view.php
      • Deze pagina geeft een overzicht van het e-mailadres, het paswoord en laatste login_time van de geselecteerde gebruiker
      • Zorg ervoor dat wanneer je naar een pagina surft je dus een correcte HTML-pagina te zien krijgt met een html-tag, head-tag, body tag die correct worden afgesloten.
      • Kijk je header en footer dus goed na.
